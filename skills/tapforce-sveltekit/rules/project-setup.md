# Project Setup Rules

## CLI-Based Project Creation

**ALWAYS** use SvelteKit CLI to create new projects. NEVER manually create core files.

### sv create Command

The primary command for creating new SvelteKit projects:

```bash
npx sv create [options] [path]
```

### Required Commands

For new SvelteKit projects:
```bash
# Basic setup
<package-manager> dlx sv create . --install <package-manager>
<package-manager> run dev

# With TypeScript and add-ons
<package-manager> dlx sv create . --types ts --add <add-ons> --install <package-manager>
<package-manager> run dev
```

### Template Options

Choose from these templates:
- `minimal`: Barebones scaffolding for your new app
- `demo`: Showcase app with a word guessing game that works without JavaScript
- `library`: Template for a Svelte library, set up with `svelte-package`

### Common Add-ons

Popular add-ons that can be added during creation:
- `tailwindcss`: Tailwind CSS v4 integration
- `eslint`: Code linting setup
- `prettier`: Code formatting setup
- `vitest`: Unit testing framework
- `playwright`: End-to-end testing
- `mdsvex`: Markdown support

### Package Manager Selection

- Use `pnpm` as default package manager (recommended)
- If project specifies different package manager, use that instead
- Supported: `npm`, `pnpm`, `yarn`, `bun`, `deno`
- Replace any `<package-manager>` placeholder with actual package manager

### Files to Never Create Manually

DO NOT manually create these files in new projects:
- `package.json`
- `tsconfig.json`
- `svelte.config.js`
- `vite.config.js`
- `.gitignore`
- Other core configuration files

These are automatically generated by SvelteKit CLI.

### Project Initialization Flow

1. Navigate to project directory
2. Run `sv create .` with appropriate package manager
3. Install dependencies with package manager
4. Start development server
5. Update project metadata in generated files

### Correct Usage Examples

```bash
# Using pnpm (recommended)
pnpm dlx sv create . --install pnpm
pnpm install
pnpm run dev

# Using pnpm with TypeScript and Tailwind CSS
pnpm dlx sv create . --types ts --add tailwindcss --install pnpm
pnpm install
pnpm run dev

# Using pnpm with common development tools
pnpm dlx sv create . --add eslint prettier vitest --install pnpm
pnpm install
pnpm run dev

# Using npm
npm dlx sv create . --install npm
npm install
npm run dev

# Using specific template
pnpm dlx sv create . --template demo --install pnpm
pnpm install
pnpm run dev

# From playground URL
pnpm dlx sv create . --from-playground="https://svelte.dev/playground/hello-world" --install pnpm
pnpm install
pnpm run dev
```

### Incorrect Usage Examples

```bash
# WRONG - Don't create package.json manually
echo '{}' > package.json

# WRONG - Don't create tsconfig.json manually
echo '{}' > tsconfig.json

# WRONG - Don't mix package managers
npm dlx sv create .
pnpm install  # Inconsistent

# WRONG - Don't omit package manager specification
pnpm dlx sv create .  # May use wrong package manager

# WRONG - Don't create core configuration files manually
touch svelte.config.js vite.config.js
```

## Project Directory Structure

After CLI setup, you should have:
- `src/routes/` - SvelteKit routing directory
- `src/lib/` - Shared components and utilities
- `static/` - Static assets
- `package.json` - Project configuration
- `svelte.config.js` - Svelte configuration
- `vite.config.js` - Vite build configuration
- `tsconfig.json` - TypeScript configuration (if applicable)

### SvelteKit Routing Structure

```
src/routes/
├── +page.svelte          # Root page
├── about/
│   └── +page.svelte      # About page
├── blog/
│   ├── [slug]/
│   │   └── +page.svelte  # Dynamic blog post
│   └── +page.svelte      # Blog listing
└── api/
    └── posts/
        └── +server.ts    # API endpoint
```
